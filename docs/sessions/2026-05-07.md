# Session 2026-05-07 — Project goal review + ADR-021/022 accepted

## Decisions

| ID | Decision |
|----|----------|
| ADR-021 | Retrieval operation contract. Public operations are legal-intent-first, not retrieval-knob-first. Two operation classes: deterministic identity resolution and hybrid evidence retrieval. The engine owns operation routing (not the agent). Result envelopes carry citation, provenance, source identity `(law_id, mst, effective_date)`, effective-date metadata, hierarchy context, and required `source_content_hash`. Ambiguity / unresolved envelope is part of the contract. Mixed citation-plus-concept routing reports composition mode (`anchor-filter` vs `anchor-boost`) in diagnostics. Effective-as-of mode uses ADR-013 temporal predicates; head/future-effective mode must be explicitly requested. Phase-1 corpus boundary: `law_id` outside SAPA + OSH returns `unresolved` with reason code `out-of-corpus`. Bounded evidence list, deduplicated to one evidence object per `(source table, source row, source_content_hash)`. Retrieval-internal signals returned in a positionally fixed `diagnostics` block, non-authoritative and additive. |
| ADR-022 | Graph modeling roadmap. ERD remains canonical normalized storage; the graph is a derived projection layer. Roadmap = Layers 1–4 (structural / reference / definition / legal semantic), implemented by Graph Stages 1–4 respectively. Phase-1 graph projection is an identity and traversal contract over existing ERD rows — no new graph tables, SQL views, materialized views, external graph stores, projection jobs, or refresh scheduling. Bounded by ADR-019 SAPA + OSH corpus and ADR-013 effective-as-of semantics. Each legal document is its own structural root; Act-to-Decree linkage is not Layer 1 containment (points to ADR-009 or its follow-up). Layer 1 node identity is structural, version-scoped to `(law_id, mst, effective_date)`, source-kind deterministic, not content-hash-derived. Source-kind identity inputs: `structure_nodes.node_key`, annex / supplementary-provision / form source-row identifiers. Document package picture under one legal document root: structural text hierarchy, supplementary provisions, annexes, forms — package-local ordering, not one merged document-wide chain. Annexes are document-level legal content units (per ADR-001 not article children/siblings). `annex_attachments` and `form_attachments` are resources/provenance, not primary Phase-1 graph nodes. Layer 2 must support typed unresolved / out-of-corpus references; vocabulary deferred. Layer 4 is qualitatively different from Layers 1–3 (closer to legal ontology / NLI research) and requires significant scope reduction or external collaboration before commitment. |

## Project goal review (durable position)

> This project is a production-grade Korean legal retrieval engine, not a
> chatbot and not a full RAG system.

Durable goal: prove that a direct, measurable retrieval engine can
outperform or complement thin 법제처 API wrappers by

- preserving source-native legal structure in separate source ERDs;
- building a unified `chunks` search layer with real source FKs;
- supporting authority, provenance, license, and temporal metadata;
- quantitatively evaluating Korean legal retrieval quality with Recall@K
  and MRR;
- comparing hybrid retrieval against weaker or narrower baselines;
- keeping generation, multi-agent routing, and orchestration frameworks
  out until retrieval quality is measured.

Phase-1 goal: build the SAPA + OSH statutory baseline from canonical
`eflaw` source rows through the first hybrid retrieval evaluation.

Phase-1 next-meaningful proof point is **retrieval**:

- generate chunks from `structure_nodes` and `annexes`;
- default retrieval/eval to the legally effective slice as of `2026-05-06`;
- require explicit query mode for head/future-effective slices;
- BM25 + vector candidate retrieval, RRF fusion (original query 2x
  weight), `bge-reranker-v2-m3` reranker;
- compare `bge-m3` and KURE before freezing the embedding model.

Out of scope: generation, LangChain / LlamaIndex, non-statutory Phase-1
expansion before measurement, LLM router / multi-agent workflow.

## Accepted Phase-1 implications

From ADR-021:

- Retrieval baseline must distinguish deterministic citation /
  identifier resolution from hybrid evidence retrieval.
- Public operation inputs are legal-intent-first.
- Valid results carry citation, provenance, source identity,
  effective-date metadata, hierarchy context, and source hash.
- Deterministic ambiguity / unresolved results and mixed
  citation-plus-concept routing are part of the contract.
- Deterministic resolution covers `structure_nodes`, `annexes`,
  `supplementary_provisions`, `forms`, and binary attachments. The
  chunks index itself remains restricted to ADR-003 Phase-1 source FKs.

From ADR-022:

- Phase-1 graph work is a derived Layer 1 structural projection over
  existing ERD rows, not new graph storage.
- Layer 1 node identity is structural, version-scoped, source-kind
  deterministic, not content-hash-derived.
- `source_content_hash` per ADR-021 tracks row content within a node;
  it is not the node's identity.
- Document package shape, package-local ordering, and Act-to-Decree
  non-containment are accepted roadmap constraints.

## Future requirements recorded (not Phase-1)

- **RNN** is a future engine requirement; explicitly out of scope for
  Phase-1. Acronym expansion and role remain `<TBD>`. Extensibility
  must still be considered now, but RNN-specific contract design is
  premature. ADR-021 expresses the underlying generic constraint
  (retrieval operations remain composable with future downstream
  stages; evidence objects are machine-structured; internal diagnostics
  are not public knobs) without binding to an RNN-specific shape.
