## Empirical findings worth preserving

- `lawService.do?target=law&MST=271485` returns OSH Rule XML with
  `<법종구분>고용노동부령</법종구분>` and `<시행일자>20250601</시행일자>`;
  adding `efYd=20260101` to `target=law` does not change that XML.
- `lawService.do?target=eflaw` returns distinct XML for the same `MST`
  when `efYd` changes (verified across OSH Act `MST=283449`, OSH Decree
  `MST=263363`, OSH Rule `MST=263749`, OSH Rule `MST=271485`). This
  triggered ADR-019's "stop and draft a raw-retention identity ADR"
  clause and produced ADR-020.
- Canonical `eflaw` file hashes differ from legacy `target=law` file
  hashes; legacy DB rows must not be silently updated in place. Clean
  rebuild or explicit replacement step required.
