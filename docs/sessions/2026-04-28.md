# Session 2026-04-28 — TODO-3 cleanup; ADR-004/005/006/007 accepted

## What happened

Long session, four ADRs cleared in one run. The ADR-004→005 and
ADR-006→007 pairs each followed a "primary decision → downstream
question surfaced inside the primary's Out-of-scope list →
follow-up ADR closes it" shape, which kept the cluster internally
coherent.

1. **TODO-9 → ADR-004 accepted** — `supplementary_provisions`
   stays a separate table, not merged into `structure_nodes` under
   a sentinel `level`. The merge variant would force
   `부칙공포일자` and `부칙공포번호` nullable on every
   `structure_nodes` row — the same polymorphic-table pattern
   ADR-003 rejected on the chunks side. Decision is zero-rewrite;
   ERD draft, ADR-002, and ADR-003 already reflect the
   separate-table shape. Three downstream questions explicitly
   deferred. Commit `92072b1`.

2. **ADR-005 accepted** — `supplementary_provisions` is **not** a
   chunk source in Phase 1. Closes ADR-004's deferred Q1 and
   flushes ADR-003's unstated assumption (the omission of
   `provision_id` in the chunks DDL was a live default without an
   argument). ADR-005 makes that omission load-bearing. Reasoning
   centered on cross-law spurious matches in 일부개정 부칙 (e.g.,
   도서관법, 화학물질관리법) and the 35804/35805 near-duplicate
   pair that would produce identical-text chunks if indexed.
   Trade-off accepted: phased-enforcement carve-out queries (e.g.,
   "50인 미만 적용 시기") cannot be answered via free-text search
   in Phase 1; SQL-queryable from `supplementary_provisions.content`.
   Three explicit revisit triggers documented. Commit `08901b2`.

3. **TODO-3 cleanup + TODO-6 → ADR-006 accepted** — Phase-1 ERD
   scope marked RESOLVED (Option B; effectively Option A for this
   statute since 중대재해처벌법 has no 시행규칙). No ADR — the
   choice was forced by empirical reality, not a real decision.
   Recorded for TODO-list hygiene. Then ADR-006 accepted:
   `doc_type` is `TEXT + CHECK` over the formal Korean type set
   {법률, 대통령령, 총리령, 부령}; `level` is `SMALLINT + CHECK`
   over 1..8 mapped to 편→장→절→관→조→항→호→목. PG ENUM rejected
   on `ALTER TYPE` caveats and ORM friction; reference tables
   rejected as overkill for closed taxonomies with no per-row
   metadata. **Material correction surfaced**: the prior ERD
   draft used informal role names (시행령, 시행규칙). The API
   returns formal type names. Two acceptance refinements: DDL
   convention (named constraints land inline on `CREATE TABLE`,
   not as follow-up `ALTER TABLE`) and an inferred-value
   verification trigger for ministry-prefixed variants like
   `'행정안전부령'`. Commit `6ebde11`.

4. **ADR-007 accepted** — `doc_type_code TEXT NULL` (no CHECK)
   captured as a sibling column on `legal_documents`. Closes
   ADR-006's "Still open" #2. Companion to `doc_type`: human-facing
   canonical (`doc_type`) + machine-facing stable identifier
   (`doc_type_code` = e.g. `A0002`, `A0007`); both populated, not
   redundant. Reasoning: code is already on the parse path,
   backfill is asymmetrically expensive, provenance is evaluation
   infrastructure for a retrieval engine, and the column hedges
   ADR-006's accepted residual risk near-cost-free. Nullable + no
   CHECK because the role is provenance capture, not constraint
   enforcement (`doc_type`'s CHECK plays the constraint role).
   Commit `f355e4b`.

## Patterns worth flagging

1. **Pre-draft empirical verification surfaces material
   corrections.** ADR-006's drafting verified that the API returns
   formal type names (`대통령령`) while the ERD draft was using
   informal role names (`시행령`). That kind of correction
   doesn't come out of memory; it comes out of re-reading the XML
   under the ADR's specific question. Worth keeping as a habit
   for future ADRs.
2. **Out-of-scope lists in ADRs propagate cleanly into follow-up
   ADRs.** ADR-004's "Out of scope #1" became ADR-005's
   "Resolves." ADR-006's "Still open #2" became ADR-007's
   "Resolves." This is the working version of §5b — the
   editable surface stays the ADR, scope creep is contained, and
   the cluster gains internal consistency over time.
3. **Devil's Advocation pass was light this session.** Each ADR
   approved with refinements or unchanged; no full rewrites. Two
   non-exclusive readings: (a) ADRs are landing well-pre-DA'd
   (each pre-DA self-check inside the draft did its job); or
   (b) the decision-shapes today were genuinely simpler than
   ADR-001/002/003. Probably some of both. Worth re-checking
   the next time a meatier decision (e.g., TODO-5 amendment
   policy) is in flight.
