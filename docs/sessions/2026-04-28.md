## Empirical findings worth preserving

- 일부개정 부칙 across statutes (e.g., 도서관법, 화학물질관리법) and the
  35804/35805 near-duplicate pair would produce cross-law spurious matches
  if `supplementary_provisions` were indexed as chunks. Drives ADR-005.
- Pre-draft empirical re-read of XML during ADR-006 caught that the API
  returns formal type names (`대통령령`), while the prior ERD draft used
  informal role names (`시행령`). Material correction; future ADRs should
  re-verify against XML under the ADR's specific question.

## Project conventions established

- Out-of-scope lists in ADRs propagate cleanly into follow-up ADRs
  (ADR-004 #1 → ADR-005 "Resolves"; ADR-006 #2 → ADR-007 "Resolves").
- DDL convention: named constraints land inline on `CREATE TABLE`, not as
  follow-up `ALTER TABLE`.
- CHECK-over-TEXT representation pattern is now project default for closed
  taxonomies with no per-row metadata (PG ENUM rejected on `ALTER TYPE`
  caveats; reference tables rejected as overkill).
